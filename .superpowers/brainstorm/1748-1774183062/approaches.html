Три подхода к реализации

Выбери архитектуру портала — каждый вариант имеет свои плюсы и минусы

A

React + Vite + Supabase · PWA ⭐ Рекомендую

Стек: React 18, Vite, Supabase (auth + DB), GitHub Pages + GitHub Actions (auto-build)

Плюсы:

  • ✅ Устанавливается на iPhone как приложение (Add to Home Screen)
  • ✅ Push-уведомления таймера работают даже при свёрнутом экране
  • ✅ Offline режим — видишь программу без интернета
  • ✅ Компонентная архитектура — легко масштабировать
  • ✅ Богатая экосистема: charts, анимации, UI-библиотеки
  • ✅ Supabase бесплатный tier более чем достаточен (500MB, 50K MAU)

Минусы:

  • ⚠️ Нужен build step (GitHub Actions — настраивается один раз)
  • ⚠️ Чуть сложнее старта, чем vanilla

Хостинг: GitHub Pages (бесплатно) · БД: Supabase (бесплатно)

B

Vanilla JS + Supabase · PWA

Стек: HTML/CSS/JS чистый, Supabase JS SDK, GitHub Pages (прямой деплой без сборки)

Плюсы:

  • ✅ Максимально простой деплой — push в репо = сайт обновлён
  • ✅ Нет зависимостей от фреймворков, нет npm install
  • ✅ PWA и уведомления работают так же
  • ✅ Молниеносная загрузка

Минусы:

  • ❌ Без компонентов — код быстро станет громоздким при расширении
  • ❌ Сложнее поддерживать сложную логику программы тренировок
  • ❌ Нет типобезопасности — баги сложнее отловить

Хостинг: GitHub Pages (бесплатно) · БД: Supabase (бесплатно)

C

Next.js + Supabase · Vercel

Стек: Next.js 15, Supabase, Vercel hosting

Плюсы:

  • ✅ Server-side rendering — быстрый первый экран
  • ✅ API Routes для сложной логики программ (без отдельного бэкенда)
  • ✅ Vercel деплой в 1 клик, очень удобно
  • ✅ PWA поддержка через next-pwa

Минусы:

  • ⚠️ Vercel free tier ограничен (100GB bandwidth/мес — хватит для личного использования)
  • ⚠️ Сложнее чем React+Vite для этого масштаба приложения
  • ⚠️ Избыточность: SSR не нужен для личного приложения

Хостинг: Vercel (бесплатно для личного) · БД: Supabase (бесплатно)

💡 Моя рекомендация: Вариант A

React + Vite + Supabase + GitHub Pages — лучший баланс между возможностями и простотой поддержки. PWA даёт нативные уведомления таймера на iPhone. Один раз настраиваем GitHub Actions и дальше деплой автоматический при каждом push.
