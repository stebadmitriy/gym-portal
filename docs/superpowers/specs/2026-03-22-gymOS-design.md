# GymOS — Design Specification
**Date:** 2026-03-22
**Author:** brainstorming session
**Status:** Approved by user

---

## 1. Overview

GymOS — персональный тренировочный портал для одного пользователя. Цель: прогрессировать в нагрузках и снизить процент жира за 6 месяцев. Пользователь тренируется 2 раза в неделю в зале Prime Gym + 15K шагов ежедневно.

**Текущее состояние пользователя:** небольшой слой жира на животе/груди, хорошая база после 1.5 лет тренировок, уверенный средний уровень.
**Цель через 6 месяцев:** V-тейп, широкие плечи, рельефный торс, минус жир.

## 2. Tech Stack

| Компонент | Решение |
|---|---|
| Frontend | React 18 + Vite |
| Стиль | Tailwind CSS + кастомная тема Gradient Energy |
| БД | Supabase (PostgreSQL) — бесплатный tier |
| Авторизация | Supabase Auth — PIN-защита (простой пароль) |
| Хостинг | GitHub Pages |
| CI/CD | GitHub Actions (auto-build при push) |
| PWA | vite-plugin-pwa — Service Worker, Web Push API |
| Графики | Recharts |
| Анимации | Framer Motion |
| Видео | ExerciseDB API (GIF) + Instagram oEmbed |

**Ключевое:** приложение устанавливается на iPhone через Safari → Add to Home Screen. Push-уведомления таймера работают на iOS 16.4+ (iPhone 17 — без проблем).

## 3. Экраны и навигация

Нижняя навигация (Bottom Tab Bar) — 5 вкладок:

```
🏠 Главная | 🔥 Тренировка | 📈 Прогресс | 🥗 Питание | 📚 Упражнения
```

Дополнительно: ⚙️ Настройки (из Главной).

### 3.1 Главная
- Карточка следующей тренировки (название, день, длительность, кол-во упражнений)
- Прогресс-бар текущего блока (неделя X из 4)
- Кнопка "Начать тренировку"
- Шаги сегодня (ввод вручную, прогресс к 15K)
- Питание дня (ккал/белок/дефицит — краткая сводка)
- Последний результат (лучшее упражнение прошлой тренировки)

### 3.2 Активная тренировка
- Таймер общей длительности тренировки (в статус-баре)
- **Таймер отдыха:** большой countdown, уведомление за 30 сек до конца
- Карточка текущего упражнения:
  - Название + мышцы
  - Вес (кнопки −2.5 / +2.5 кг)
  - Счётчик подходов (визуальные точки: ✓ / активный / pending)
  - Количество повторений
- **Прогресс-бар тренировки** (полоска + чипы с названиями упражнений)
- **Свайп-шторка вверх** — полный список упражнений:
  - ✅ зелёный = выполнено
  - 🟠 оранжевый = текущее
  - ⬜ серый = впереди
  - Счётчик подходов для каждого
- После тренировки: быстрая оценка 😓 / 👍 / 💪 → влияет на следующий вес

### 3.3 Прогресс
- Выбор упражнения → график веса по неделям (Recharts line chart)
- Замеры тела: вес, обхват талии, дата добавления
- График замеров по времени
- Сравнение: текущий блок vs предыдущий цикл

### 3.4 Питание
- Калькулятор при первом запуске: вес, рост, возраст, уровень активности → TDEE
- Дневная норма: ккал, белок (г), дефицит (-400 ккал)
- В день тренировки: +200 ккал к норме (автоматически)
- Советы дня (статичный набор из ~30 ротируемых советов)
- Ручной трекер приёма пищи: **не входит в v1**. Экран питания показывает только расчётные нормы и советы.

### 3.5 Упражнения
- Список всех упражнений программы (тренажёры Prime Gym)
- Фильтр по группе мышц
- Карточка упражнения:
  - GIF анимация (ExerciseDB API)
  - Целевые мышцы (primary + secondary)
  - Советы тренера — 3-4 пункта на русском (захардкоженные)
  - Instagram embed (@appyoucan) через oEmbed — Reel с техникой
  - YouTube embed — дополнительное видео с разбором

### 3.6 Настройки
- Профиль: имя, вес, рост, дата старта
- PIN-код (4 цифры). Хранится как bcrypt-хэш в `users.pin_hash`. Сравнение через Supabase Edge Function (не на клиенте). При входе: клиент отправляет PIN → Edge Function сравнивает bcrypt → возвращает JWT сессию.
- Таймер отдыха: настройка длительности (60/90/120/180 сек)
- Уведомление за X сек (30 по умолчанию)

## 4. Программа тренировок — алгоритм

### 4.1 Структура тренировок

**Full Body 2x в неделю** (научное обоснование: при 2 тренировках в неделю Full Body даёт вдвое больше жиросжигания и равный мышечный рост vs split).

**Тренировка A (акцент — толкание + тяга горизонт):**
1. Chest Press (Жим грудных) — 4×10-12
2. Lat Pulldown / Linear Back Row (Тяга верхнего блока) — 4×10-12
3. Shoulder Press (Жим плечей) — 3×10-12
4. Leg Press (Жим ногами) — 4×10-12
5. Seated Leg Curl (Сгибания ног) — 3×12
6. Biceps Curl (Бицепс) — 3×12
7. Abdominal Crunch (Пресс) — 3×15

**Тренировка B (акцент — тяга вертикаль + изоляция):**
1. Pull Over (Пуловер) — 4×10-12
2. Pec Fly / Incline Pec Fly (Разводка) — 4×10-12
3. Low Row (Горизонтальная тяга) — 4×10-12
4. Leg Extension (Разгибания ног) — 4×12
5. Iso Kneeling Hamstring Curls — 3×12
6. Assist Dip/Chin (Отжимания/подтягивания) — 3×max
7. Abdominal Crunch — 3×15

### 4.2 Блочная периодизация (старт с Блока 2)

| Блок | Недели | Повторения | % 1ПМ | Подходы |
|---|---|---|---|---|
| **Блок 2: Гипертрофия** | 1–4 | 10–12 | 70–80% | 4 |
| **Блок 3: Сила** | 5–8 | 6–8 | 80–85% | 4–5 |
| **Разгрузка** | 9 | 12–15 | 50% объёма | 3 |
| **Цикл 2: Гипертрофия** | 10–13 | 10–12 | выше базы | 4 |
| **Цикл 2: Сила** | 14–17 | 6–8 | ещё выше | 4–5 |
| **Разгрузка** | 18 | — | 50% | 3 |

**Блок 1 (Накопление)** используется только как возврат после болезни/пропуска 2+ недель.

### 4.3 Прогрессия весов

- Базовый шаг: **+2.5 кг** в неделю на основных упражнениях
- Оценка после тренировки:
  - 😓 Тяжело → следующая неделя: −5% от текущего веса
  - 👍 Нормально → +2.5 кг (план)
  - 💪 Легко → +5 кг вместо +2.5 кг

**Приоритет ручной коррекции:**
- Если пользователь изменил вес вручную во время тренировки (кнопки −/+), это фактический выполненный вес записывается в `workout_sets.weight_kg` и становится **новым базовым весом** для алгоритма прогрессии.
- Оценка тренировки (easy/normal/hard) применяется поверх этого нового базового веса (не поверх запланированного веса).
- Пример: план был 75 кг, пользователь снизил до 70 кг → base = 70 кг. Оценил "нормально" → следующая неделя = 72.5 кг.

### 4.4 Тренажёры Prime Gym (полный список)

**Ноги:** Leg Press, Super Squat, Seated Leg Press, Leg Extension, Heavy Seated Leg Extension, Leg Curl (lying), Seated Leg Curl, Rear Kick, Iso Kneeling Hamstring Curls, Leg Swing, Abductor/Adductor, Abduction

**Спина:** Lat Pulldown (Pull Down), Linear Back Row, Low Row, Pull Over, Dual Cable Cross (12 вариантов)

**Грудь:** Chest Press, Pearl Delt/Pec Fly, Incline Pec Fly, Incline Olympic Bench, Olympic Decline Bench

**Плечи:** Shoulder Press

**Руки:** Biceps Curl, Assist Dip/Chin

**Кор:** Abdominal Crunch

**Свободные веса:** Smith Machine (×2), T-bar Row / Landmine

**Кардио:** Rowing Machine, Ski Erg/Climber

## 5. База данных (Supabase)

### Таблицы

```sql
-- Профиль пользователя
users (
  id, name, weight_kg, height_cm, age,
  start_date, pin_hash, rest_timer_seconds,
  notification_before_seconds
)

-- Программа тренировок (seed-таблица, заполняется при инициализации)
-- Структура A/B тренировок и упражнения по блокам хардкожены в коде (src/data/program.ts)
-- и при первом запуске сидируются в эту таблицу для удобства запросов.
-- Frontend запрашивает план по: workout_type + block_number → получает список exercise_id + целевые сеты/повторения.
workout_plans (
  id, workout_type (A/B), block_number (2/3),
  week_in_block (1-4), exercise_id,
  target_sets, target_reps_min, target_reps_max,
  sort_order
)

-- Упражнения (справочник)
exercises (
  id, name_ru, name_en, machine_name,
  muscle_primary, muscle_secondary,
  tips_ru (jsonb array),
  gif_url (ExerciseDB),
  instagram_reel_url,
  youtube_url
)

-- Тренировочные сессии
workout_sessions (
  id, user_id, date, workout_type,
  block, week, duration_minutes,
  rating (easy/normal/hard),
  notes
)

-- Подходы (каждый выполненный сет)
workout_sets (
  id, session_id, exercise_id,
  set_number, weight_kg, reps,
  completed_at
)

-- Замеры тела
body_measurements (
  id, user_id, date,
  weight_kg, waist_cm, notes
)

-- Шаги (цель всегда 15000 — константа в коде, не хранится в БД)
daily_steps (
  id, user_id, date, steps
)
```

## 8. Питание — алгоритм

**TDEE расчёт (Harris-Benedict + коэффициент активности):**
- Базовый метаболизм (BMR) по весу/росту/возрасту
- Коэффициент: 1.375 (умеренная активность — 2 тренировки + ходьба)
- Дефицит: −400 ккал/день (научно оптимальный для сохранения мышц)
- В день тренировки: +200 ккал
- Белок: 1.8–2.0 г/кг веса тела (цель >1.3г/кг минимум)

## 9. Объём работы / что НЕ входит в v1

**Входит:**
- Все 6 экранов
- Алгоритм программы (2 тренировки, блочная периодизация)
- Таймер отдыха с Push-уведомлениями
- Прогресс-бар + свайп-шторка
- Графики прогресса (Recharts)
- Замеры тела
- Калькулятор TDEE + питание
- Библиотека упражнений Prime Gym с GIF + Instagram embed
- PWA (Add to Home Screen на iPhone)
- PIN-защита через Supabase Auth
- GitHub Pages деплой + GitHub Actions

**НЕ входит в v1:**
- Apple Health интеграция
- Автоматический подсчёт шагов
- AI-генерация программы
- Многопользовательский режим
- Социальные функции

## 10. Решения по видео-контенту

- **GIF техники:** ExerciseDB API (бесплатно, ~1300 упражнений)
- **Instagram @appyoucan:** официальный oEmbed (встраивание публичных Reels). Требует авторизации в Instagram для просмотра.
- **YouTube:** embed для дополнительных видео-разборов
- **Советы тренера:** захардкоженные тексты на русском, 3-4 пункта на упражнение
